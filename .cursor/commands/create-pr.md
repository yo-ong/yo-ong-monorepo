# Create PR

## Overview
변경사항을 잘 정리한 PR을 작성합니다.

## Steps
1. **브랜치 준비**
   - 변경사항 커밋
   - 원격 브랜치 푸시
   - main과 동기화 확인

2. **PR 설명 작성**
   - 변경 내용 요약
   - 맥락과 동기
   - Breaking change 여부
   - UI 변경 시 스크린샷

3. **PR 설정**
   - 명확한 제목
   - 적절한 라벨
   - 리뷰어 지정
   - 관련 이슈 링크

## Checklist
- [ ] pnpm build 성공
- [ ] pnpm lint 통과
- [ ] pnpm tsc 통과
